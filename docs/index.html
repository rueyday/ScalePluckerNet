ScalePluckerNet

Extending 3D line registration from SE(3) to Sim(3) — jointly recovering rotation, translation, and scale from Plücker line correspondences.

Ruey Day

Department of Robotics · University of Michigan, Ann Arbor · 2026

Code Built on PlueckerNet (Liu et al., CVPR 2021)

Abstract

PlueckerNet (Liu et al., CVPR 2021) registers 3D line reconstructions under rigid SE(3) transformations. We show that its internal solver structurally fails when an unknown scale factor is present, as in monocular camera systems — the moment term transforms as m′ = s·Rm + t×d′ under Sim(3), and the SE(3) solver misinterprets the residual (s−1)·Rm as a spurious translation.

We address this with a minimal two-correspondence Sim(3) solver — SVD for rotation, then a joint least-squares solve for scale and translation — and train the unmodified PluckerNetKnn architecture on the Replica RGBD benchmark (7 scenes, 4200 pairs). On Chess 7-Scenes with simulated monocular scale (s = 1.8), ScalePluckerNet achieves 5.75° rotation error and scale-log-error 0.149, vs. 19.02° / 0.588 for SE(3)-PlueckerNet. A 9D extension appends per-line RGB (with 30% dropout) and reaches 99.4% inlier ratio on validation.

Failure Analysis

Analytical proof + experiments showing the SE(3) Plücker solver fails structurally under Sim(3): correct rotation, biased translation.

⚙️

Sim(3) RANSAC Solver

Minimal 2-correspondence solver: SVD on direction pairs for R, then a 6×4 least-squares system for (s, t) jointly.

🏛️

Real-Data Training

Trained on Replica RGBD geometry (7 scenes, 4200 pairs) — closes the 89° synthetic-to-real gap to 9.4° on Chess.

9D Color Extension

Appends per-line RGB to 6D Plücker coordinates; 30% color dropout enables a single model to handle RGBD, RGB-only, and mixed inputs.

Method

The PluckerNetKnn architecture is unchanged — only the RANSAC back-end is extended from SE(3) to Sim(3).

// input: two Plücker line clouds [m, d] (moment-first) Cloud 1 [m,d] (N₁×6) ──────┐ ┌────── Cloud 2 [m,d] (N₂×6) │ │ ▼ ▼ ┌─── PluckerNetKnn (unchanged) ───┐ │ KNN graph conv (moments) │ │ KNN graph conv (directions) │ │ Spatial Attentional GNN (×6) │ │ Sinkhorn OT (30 iters) │ └──────────────┬───────────────────┘ │ prob_matrix ∈ ℝᴺ¹ˣᴺ² ▼ Top-K select │ K matched Plücker pairs ▼ ┌──── Sim(3) RANSAC (new) ────────┐ │ Stage 1: R ← SVD on d-pairs │ │ Stage 2: [s,t] ← lstsq 6×4 │ │ Score: ‖L₂ − M_sim3·L₁‖ < τ │ └──────────────┬───────────────────┘ ▼ s, R, t

Minimal Solver — 2 line pairs

Given R (from direction SVD, scale-invariant), each line contributes one moment equation:

s·Rm₁ᵢ − [d₂ᵢ×]·t = m₂ᵢ Stack 2 pairs → 6 equations, 4 unknowns [s, t]: ⎡ Rm₁₁ −skew(d₂₁) ⎤ ⎡ s ⎤ ⎡ m₂₁ ⎤ ⎢ ⎥ ⎢ ⎥ = ⎢ ⎥ → x = lstsq(A, b) ⎣ Rm₁₂ −skew(d₂₂) ⎦ ⎣ t ⎦ ⎣ m₂₂ ⎦ Hypotheses with s ≤ 0 are rejected.
